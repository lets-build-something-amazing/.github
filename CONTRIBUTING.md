# **Contributing to Let's Build Something Amazing (LBSA)**

First off, thank you for considering contributing to LBSA\! 👋 🌎 → 🌱 → 🚀

We are an open-source engineering community built to bridge the gap between beginner tutorials and real-world collaboration. Whether you are writing your first line of code, designing architecture, or simply suggesting a great idea, your contribution is valuable here.

## **🟢 Current Status: Phase 1 (The Gathering)**

**Important Note:** We are currently in **Phase 1: The Gathering**. This means we are focused on building our community and brainstorming ideas.

**We are not currently accepting code pull requests for a main application.**

However, you can still contribute right now\! Here is how:

### **1\. Pitch an Idea (The Civic Challenge)**

We are looking for project ideas that leverage open-source technology to create a positive **civic impact** in our local communities.

* Head over to our [Discord Server](https://letsbuildsomethingamazing.com).  
* Go to the \#what-are-we-building forum.  
* Create a new post using the provided pitch template.

### **2\. Improve Our Documentation**

Notice a typo in this file? Have an idea to make our GitHub organization front page more welcoming? We *are* accepting Pull Requests for documentation and repository structure\!

## **🛠️ How to Contribute (When Phase 3 Begins)**

Once we vote on a project and enter **Phase 3: The Build**, these will be our standard operating procedures.

### **The "Zero-Cost / BYOK" Rule**

We strictly utilize GitHub Codespaces / DevContainers. You are never required to pay for hosting, compute, or local setup software to contribute to LBSA.

* We embrace AI-assisted coding, but we operate on a **Bring Your Own Key (BYOK)** model. You provide your own API keys for your preferred AI tools within your isolated Codespace.

### **1\. Branching Strategy**

We follow a standard Git Flow. **Never commit directly to main.**

* Create a new branch from main for your work.  
* **Include the Issue number** in your branch name. This helps mentors instantly understand what you are working on so they can proactively help\!  
* Naming convention: feature/\[issue-number\]-short-description (e.g., feature/42-add-dark-mode), bugfix/\[issue-number\]-description (e.g., bugfix/115-fix-header-alignment), or docs/update-readme.

### **2\. Submitting a Pull Request (PR)**

* Ensure your code runs successfully in the provided DevContainer environment.  
* Keep your PRs small and focused on a single issue.  
* Provide a clear and detailed description in your PR. If it fixes an open issue, link it (e.g., "Fixes \#42").  
* Wait for a maintainer or senior contributor to review your code. We prioritize mentorship, so expect constructive feedback\!

## **🤝 Code of Conduct**

By participating in this project, you agree to abide by our Code of Conduct (available in our Discord server). We are dedicated to providing a welcoming, harassment-free experience for everyone, regardless of experience level.
